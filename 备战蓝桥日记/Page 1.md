### 新栏目，备战蓝桥杯的点点滴滴

今天做到一道题目感觉挺有意思的，先看看题干

>   ## 题目描述
>
>   给定一个整数，请将该数各个位上数字反转得到一个新数。新数也应满足整数的常见形式，即除非给定的原数为零，否则反转后得到的新数的最高位数字不应为零（参见样例2）。
>
>   ## 输入格式
>
>   一个整数 N*N*
>
>   ## 输出格式
>
>   一个整数，表示反转后的新数。
>
>   ## 输入输出样例
>
>   **输入 #1**
>
>   ```
>   123
>   ```
>
>   **输出 #1**
>
>   ```
>   321
>   ```
>
>   **输入 #2**
>
>   ```
>   -380
>   ```
>
>   **输出 #2**
>
>   ```
>   -83
>   ```

就是一个普通的反转数字，题目中给的N的范围是正负1e9,所以我首先想的是先从数字的末尾下手，因为不知道整个数字的长度。要是使用int的话，从末尾下手比较方便，所以就写出了初版

```cpp
#include<iostream>
#include<cstdio>
#include<queue>
using namespace std;

int main() {
    int n;
    queue<int>s;
    cin >> n;
    bool flag;
    if(n<0) {
        flag = true;
        n = -n;
    }
    else flag = false;
    while(n!=0){
        if(n%10!=0) {
            s.push(n%10);
            n/=10;
        }
        else n/=10;
    }
    if(flag){
        cout << "-";
    }
    while(!s.empty()){
        cout << s.front();
        s.pop();
    }
    return 0;
}
```

但当我提交了代码之后，我发现Unaccepted了，只有70分，里面有些点没过，然后就跑去看一下到底是哪些数据没过。

根据题目的要求，反转后最高数位不能是0，所以第一版中计算部分对0的处理太过粗糙了。这里分成两种情况，一种是输入的数据本身就是0，我们直接输出0就行了；第二种就是输入的数据如10001这样的情况，如果用第一版的代码只能输出11这样的数据，所以得进行一下讨论。

```cpp
#include<iostream>
#include<cstdio>
#include<queue>
using namespace std;

int main() {
    int n;
    queue<int>s;
    cin >> n;
    if(n == 0){ cout << "0"; exit(0);} 
    bool flag, idx = true;
    if(n<0) {
        flag = true;
        n = -n;
    }
    else flag = false;
    while(n!=0){
        s.push(n%10);
        n/=10;
    }
    if(flag){
        cout << "-";
    }
    while(!s.empty()){
        if(s.front()==0 && idx){s.pop();}
        else{
            cout << s.front();
            s.pop();
            idx = false;}
    }
    return 0;
}
```

更改后顺利的AC了，我这里首先是加上了对0的判断，如果是0的话，直接输出0；

然后是第二种情况。

这里我分析了一下，我们需要将哪些0去掉，题目中所说的是反转过后的数字，在这个数字前面的0要删去，那么在这些0之前是没有除了0之外的数字出现的，也就是说在程序中，只要是我们输出了除了0之外的数字后，我们对0的处理就变成了只要输出即可，不需要在对其删除。

于是我这里引入了一个新的bool类型的变量，用来标记之前是否输出了0之外的数字，一旦这个idx被置为false，就可以一路畅通无阻的输出下去了。
